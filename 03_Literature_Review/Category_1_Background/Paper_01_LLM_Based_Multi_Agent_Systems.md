# Research Problem

Traditional software engineering tasks are becoming increasingly complex, and a single Large Language Model (LLM) agent cannot efficiently handle planning, collaboration, debugging, testing, and software development. There is a need for multiple AI agents that can collaborate to solve complex software engineering tasks.

---

# Methodology

The authors conducted a systematic literature review of 71 research papers related to LLM-Based Multi-Agent Systems (LMA) in Software Engineering.

The study includes:

- Literature Review
- Two Case Studies using ChatDev
- Current Capabilities Analysis
- Limitations Analysis
- Future Research Agenda

---

# Key Findings

• Multi-Agent Systems improve software development.

• Different AI agents can perform specialized roles such as Programmer, Reviewer, Tester, Architect, Product Manager, and QA Engineer.

• Collaboration between multiple AI agents improves software quality, robustness, and scalability.

• Multi-Agent Systems are effective for moderately complex software engineering tasks.

---

# Results

The paper concludes that LLM-Based Multi-Agent Systems can successfully automate many stages of the Software Development Life Cycle (SDLC), including:

- Requirements Engineering
- Code Generation
- Software Testing
- Debugging
- Code Review
- Software Maintenance
- End-to-End Software Development

However, current systems still struggle with highly complex reasoning tasks.

---

# Limitations

• Limited reasoning capability for highly complex software development tasks.

• Weak role specialization.

• Limited prompting strategies.

• Limited evaluation methods for collaboration.

• No dedicated framework for automatic failure recovery.

• Self-reflection and iterative repair are not fully explored.

---

# Research Gap

Although current LLM-Based Multi-Agent Systems improve collaboration and software development, there is still no dedicated framework that enables AI coding agents to automatically detect failures, perform self-reflection, and iteratively repair errors with minimal human intervention.

This creates an opportunity for further research.
