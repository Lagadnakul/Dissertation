---

# Problem Statement

Current AI coding agents are widely used, but most research evaluates them using benchmarks such as SWE-bench.

Very little research studies how these AI coding agents actually perform in real GitHub software development projects.

There is a lack of real-world evidence about:
- AI coding agent usage
- Developer productivity
- Pull Request acceptance
- Human-AI collaboration
- Trust in AI-generated code

---

# Objective

To create the first large-scale real-world dataset (AIDev) for studying how autonomous coding agents work in actual software development.

---

# Methodology

The authors created a dataset called **AIDev** by collecting GitHub Pull Requests generated by AI coding agents.

The dataset contains:

- 456,535 Pull Requests
- 61,453 GitHub repositories
- 47,303 developers

AI Coding Agents included:

- OpenAI Codex
- Claude Code
- Devin
- GitHub Copilot
- Cursor

The researchers analyzed:

- Pull Request acceptance
- Review time
- Productivity
- Human-AI collaboration
- Code quality
- AI agent behavior

---

# Key Findings

1. AI coding agents are becoming active teammates in software development.

2. AI agents generate Pull Requests much faster than humans.

3. AI-generated Pull Requests have lower acceptance rates than human-written Pull Requests.

4. Documentation tasks are one of AI's strongest areas.

5. Human developers still review most AI-generated code.

6. Current benchmarks do not fully represent real-world software engineering performance.

---

# Limitations

- The paper studies existing AI coding agents but does not improve them.

- No failure recovery mechanism is proposed.

- No self-reflection framework is introduced.

- Does not solve autonomous error recovery.

- Focuses mainly on GitHub Pull Requests.

---

# Relation to My Dissertation

This paper provides real-world evidence that AI coding agents are rapidly becoming part of software engineering.

However, it also shows that AI-generated Pull Requests are accepted less frequently than human-written Pull Requests, indicating that current AI coding agents still make mistakes.

My research aims to improve this limitation by developing a **Self-Reflection-Based Failure Recovery Framework** that helps AI coding agents detect their own mistakes, recover automatically, and generate higher-quality code before final submission.

---

# Research Gap

Existing studies analyze how AI coding agents perform in real software projects.

However, they do not provide a framework that enables AI coding agents to automatically recover from failures using self-reflection and iterative repair.

This research gap motivates my dissertation.
