# Dissertation Day 2

## Dissertation Title

A Self-Reflection-Based Failure Recovery Framework for Agentic AI Coding Systems

---

# 1. Problem Statement

AI coding agents such as Claude Code, Gemini CLI, Codex CLI, and Aider are increasingly used to assist developers in software development tasks. These systems can generate code, modify existing projects, execute commands, and debug applications.

However, when these agents encounter failures such as syntax errors, failed test cases, dependency issues, or incorrect code modifications, they often require repeated human guidance to recover successfully.

Current AI coding agents primarily focus on generating solutions but provide limited autonomous failure recovery capabilities. This increases developer effort and reduces workflow efficiency.

Therefore, there is a need for a structured framework that enables AI coding systems to detect failures, analyze their causes through self-reflection, generate improved solutions, and iteratively repair errors with minimal human intervention.

---

# 2. Research Gap

Existing research has made significant progress in AI-assisted code generation and software engineering agents.

However, most studies focus on:
- Code generation
- Task completion
- Benchmark evaluation
- Agent performance

Limited research focuses on designing a structured self-reflection-based failure recovery framework that enables AI coding agents to autonomously recover from software development failures through iterative repair.

This dissertation aims to address this gap.

---

# 3. Research Questions

RQ1

Can a structured self-reflection framework improve failure recovery in AI coding systems?

RQ2

Does iterative repair reduce human intervention during software development tasks?

RQ3

Can the proposed framework improve task completion after failures?

RQ4

What are the strengths and limitations of the proposed framework?

---

# 4. Research Contribution

This research proposes a structured failure recovery framework for AI coding systems using self-reflection and iterative repair.

Instead of only evaluating existing tools, the study introduces a methodology that aims to improve autonomous recovery from failures during software development.
